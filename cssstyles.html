Greybox Panel Style 

The greybox panel style ('graybox' works the same, as does 'greybox') is a class that can be applied to any block element. It adds a background color, box padding, and rounded corners. This is useful for offsetting text from the standard content such as for code text or instructions.

Sidebar Classes 

A sidebar is a container primarily intended for featuring additional text relevant to, but not contained within the primary content. When created in the main content area it should probably have a float-class (float-left, float-right or float-center) and a width (col-quarter, col-thirds, col-half, col-twothirds); otherwise, it will fill the entire container width. Becasue of the smaller text size it works especially well within the left and right columns (hence the name sidebar).

Text inside a sidebar should be wrapped in P tags with proper markup. Images that exceed the size of the sidebar box will be resized to fit.

<div class="sidebar">
    <p>
        This is sidebar text. Lorem ipsum dolor sit amet, consectetur adipiscing elit.
    </p>
    <p>
        See <a href="#sidebar">Sidebar</a>.
    </p>
</div>

Sidebar is a reserved class in our foundation template. Anything with this class will be converted to all caps and will not be in a box. ALL sidebar class items should be re-classes with a different panel style.

Automated Pullquotes

Enclosing text in your content with a SPAN tag with the class "pullquote" will automatically create a pullquote of the selected text. For instance, this (self referential) sentence has a pullquote span on it. The pullquote to the left is automatically generated by the standard template. Text in automatic pullquotes has all HTML tags removed and any text in parentesis is replaced by an ellipsis.

Using the "right" class with the pullquote class will float the pullquote to the right. A citation can be generated by giving your pullquote SPAN an id= attribute. Then enclose your author's name in a CITE tag and link them with a rel= attribute.

<span class="pullquote" id='samplePull'>
    For instance, this (self referential) sentence has a pullquote span on it. The 
    pullquote to the left is automatically generated by the 
    <a href="/template/standard/index.php">standard template</a>.
</span>
<cite rel='samplePull'>
    author's name
</cite>

Depricated because this is not significantly different from the primary HTML format and is not easier for non-tech users. The added functionality (parens to ellipse) is just as easily replicated by an advanced user by commenting out the unnecessary content, while a basic user would be more likely to type an ellipse in the first place. In addition it renders this styling useless for anything which actually contains parens in the full text. It's a really neat concept, and it's really cool to see, but it's not currently solving an actual user problem that I can discern.

An announcement box is intended for the right column or to be floated. It is a fixed width (167px, the same as an Inset box) box. An Announcements box can be used to create announcements manually without using the Announcements manager. Usually, however, it is easier to do it automatically. Within an announcements block only the initial H1 Tag is outside a DIV wrapper. Everything else is considered an announcement "item". An inset box is a container primarily intended for featuring photos, but which can also be used for featured text. It is a fixed width (167px - the same as the right column width) and always floats to the right. The base element can be any block element (P, DIV, BLOCKQUOTE, etc.) and then one of the column classes is applied to it. The column classes are:

  • col-quarter: 1/4 of the container width.
  • col-half: 1/2 of the container width.
  • col-thirds: 1/3 of the container width.
  • col-twothirds: 2/3 of the container width.

Some classes add padding, borders or have widths of their own that may cause some undesired wrapping or spacing. So greybox, message, error, etc. might affect the layout of the col-* styles.

Col-styles do not automatically float so applying a float style will make them slide next to each other. In the case of multiple columns it is best if you apply a consistent float to all elements (float-left, float-right e.g.) that way they all float neatly. However for more static elements you may consider floating items according to position to make the best presentation.

    <div class="col-quarter float-left">Left Column</div>

    <br class="clear"/>
</div>

When floating elements, it is helpful to do a couple of things to insure the best performance under all browsers.

  1. Wrap your floating elements in a non-floated element to make them isolated from the template element floats.
  2. Place a <br class="clear"/> element at the end of all your floated elements to clear any lingering floats that might disrupt the presentation.
  3. Padding, borders, and margins applied after the col-classes may make them extend further than the bounds of the page or make them wrap. Be aware of your box sizes

Special Links  

Special links add additional functionality when you use these classes. There are two kinds of special links at the moment: Toggle links and PopUp/PopOut Links.

Popups:

Popups open in a repositional modal window in the center of the page. They can be resized or closed using the {x} Close link in the upper right hand corner. They can point to pages outside of the USNA sites or to local documents.

PopOuts open in a seperate window that has no address bar or menu components. They can be closed normally. Typically these are reserved for PDF or External sites.

Image Styles

Images in the standard template are assumed to be "presentational" elements. That is, they are placed on the page to be seen as content. Images can however be used as "inline" elements (icons, buttons, etc.) or "navigation" (thumbnails, for example) elements.



<img src="/templates/resources/images/chapel_foot.png" alt="Dancing Zebra" />

Images do not require width, height, or border attributes. Title is optional but alt is required by the ADA. Old attributes like "align" are deprecated in HTML5 and have been replaced in our template with float classes. Height and width can be provided but bear in mind that large photos may take a while to load (especially on mobile devices) and optimized smaller-resolution images will reduce bandwitdh and increase page speed.

With just the minimum required attributes images will appear in the central content area with an off-white border and a faint grey outline. There are attributes and classes to change this appearance.

  • autofit class will reduce the size of large photos to fit the body of the image container. Show Example
  • with-border class will force the border to appear in elements where the border is normally suppressed (left and right columns for instance).
  • no-border class will suppress the border and padding
  • The attribute border="0" will also suppress the border and padding.
  • The plain class will remove the border but not the padding.
  • The tight class will remove the padding but not the border.

Fonts

Font sizes are set to be "readable" in the current browser. If you wish to add power to your text by increasing or decreasing your font sizes we have 2 "set" sizes and 2 "relative" size classes.

  • Large text is considered as large as will fit within the line-height we've set. It can look cramped.
  • Larger text can be used to emphasize words, terms, or titles without adding a header element.
  • Smaller text reduces the size of the element font to 3/4 of the current size. This is more useful in headers where it can be used with a <br/> tag to create a subtitle.
  • Small text is considered the minimum readable in the browser/device. It is not recommended to use smaller fonts than this.

Dropcaps

Applying a 'dropcap' class to a paragraph will format the first letter in a larger, left-floated style. This does not always work well, but the style is available for use. It is recommended that it be used sparingly. It works much better on longer paragraphs than shorter ones.

Miscellaneous

Horizontal Rule (HR)

<HR/> tags are styled to be different whether they are internal or external in order to match the menu styles. There is one class (fancy) specifically for HR to change its appearance. Box styles will change their appearance, but are not recommended. The width attribute can be used to set the maximum width of the line but is not required (it defaults to 100%).

An HR tag has margins above and below, in some cases it may make spacing on the page seem wide. A "tight" class will remove the margins and make the HR snug up against the elements above and below it (although they may have margins of their own).

HRs are separating elements. They should be simple and clean to match our style and not distract from the text. ALL non-standard HRs are being dropped. Custom HRs when absolutely needed should be done locally, and are discouraged.

Tables

Tables are preformatted to have colored slightly darker headers (<TH>) and visible white lines between cells. The actual color may vary in internal and public sites as there are different color schemes. Using fill-color classes this behavior can be bypassed but should not be for tabular data which is expected to be in a consistent style.

Table attributes cellspacing, cellpadding and border are not required (and are actually deprecated in HTML5). Alignment is handled via the float-classes rather than the align tag.

Tables do not automatically fill the space they are in. Instead, they collpase to fit the data. If you want your table to fill the available space, use the width attribute. In some layouts, however a 100% width will make the right edge disappear and so 97% is the maximum recommended width.

The class="stripeMe" is provided to "candy-stripe" your table automatically. This is done via javascript and so it will automatically adjust to the number of rows on your table. Add "stripeMe" to the following table.

The class="no-border" will remove the lines between the cells. Add "no-border" to the following table.

The class="plain" will remove the coloring on the headers and the lines between the cells. Note, this does not affect colors added via fill-styles or stripeMe. Add "plain" to the following table.

Classes that add color like fill-colors can change the coloration of the cells. These colors override any other colors (such as from stripeMe) that have been pre-designed. Add a new color to the following table's row headers.

Utility Classes

There are several utility classes that change the pre-defined characteristics of elements in the template.

  • tight This removes any top and bottom margins and padding.
  • plain This removes any borders, underlines on links, list-icons on list elements, and most background colors
  • no-border This removes the outline from images, the lines between cells in a table, and the underline on links (although 'plain' is preferred for that).
  • clear this clears any floats that may be causing indent or wrapping issues. Often used on <br />.
  • hideText Elements with this class are invisible on the page.
